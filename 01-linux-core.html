一、计算机的原理

1.1 什么是计算机？

  • 计算机一般称为"电脑"，通俗理解就是通电的大脑
  • 电脑：人类对计算机的最终期望，希望它可erland以向人脑一样为人们工作

1.2 计算机的五大组成部分

  • 运算器
  • 控制器
  • 存储器
  • 输入设备
  • 输出设备

1.3 CPU

  • 控制器：作用是计算机的指挥系统，负责管理计算机其他组件的工作
  • 运算器：作用包含数学运算、逻辑运算
  • 总结：控制器和运算器本质上不是硬件，是抽象的概念
  • CPU就是安装控制器和运算器的标准制作的一个硬件，所以 CPU 又被称为计算机大脑

1.4 存储器

存储器主要是：内存和硬盘

  • 内存：基于电工作，通电就能存取数据
    • 优势：存取速度很快
    • 劣势：断电之后数据会丢失
  • 硬盘（外存）：基于磁工作
    • 优势：断电数据不会丢失，可以永久保存
    • 劣势：存取速度偏慢
  • 输入设备：向计算机内输入内容（鼠标、键盘、麦克风）
  • 输出设备：计算机向外输出数据（显示器、音响、打印机）

二、操作系统

2.1 操作系统的由来

  • 不管编写什么软件，最终的目的就是操控硬件；所有的硬件设备都是需要软件进行支配
  • 实现步骤：
    • 第一步：必须要开发一个控制系统，通过控制系统来控制计算机硬件基本运行
    • 第二步：然后开发编辑工具的功能，但凡涉及到了硬件，就调用控制系统
    • 第三步：最后由操作系统来控制计算机硬件

2.2 什么是操作系统

  • 控制系统别名：操作系统（OS），是在应用软件和硬件之间的桥梁
  • 也是一个协调、管理、控制计算机硬件资源、软件资源的一个控制对象
  • 日常所使用的 windows 包括 IOS 系统或者安卓系统都是属于桌面或者个人系统
  • 服务器操作系统

2.3 Linux 系统

  • Linux 和 windows 一样都是操作系统，但是 Linux 有两层含义：
    • 一种是 Linux 编写了开源操作系统的内核
    • 第二种就是广义上的操作系统

二、磁盘介绍

2.1 什么是磁盘

  • 硬盘是电脑永久存储数据的硬件：文件、视频、音频等等
  • 每个硬盘中心都是一摞高速运转的圆盘，在圆盘上附着一圈金属颗粒，每个金属颗粒都会有自己的磁化程度，主要存储 0 和 1

2.2 磁盘物理结构

  • 盘片：硬盘一般是由一个或者多个盘片组成的，每个盘片都是可以有两面，打开一个盘片正面 0 面，背面 1 面后面盘片以此类推
  • 磁道（track）：盘片在出厂的时候被划分了多个不同直径圆环，数据就存储在这样圆环，圆环又被称为磁道
  • 扇区：硬盘出厂的时候对磁盘进行磁道划分，划分成若干个弧段，每个弧段都被称为扇区，扇区也是硬盘上的物理存储单位，每个扇区可以存储 512 字节
  • 柱面：柱面是抽象的逻辑上的概念，出于同一个垂直一个区域的磁道被称为柱面。作用：主要为了通过算法检测磁头的寻磁时间
  • 磁头：读取磁盘上金属颗粒，主要负责读取或者写入数据

2.3 磁盘的接口类型

  • IDE-SCSI：比较旧的硬盘接口
  • SATA-SAS：SATA III 和 SAS
  • MSATA-M2：MSATA 接口主要是为超极本设计的；M2 是 inter 推出的一种条带 MSTATA 接口规范

四、分区管理

4.1 为什么要分区

Linux 系统初始化的时候，跟 MBR 识别硬盘设备。

  • MBR：硬盘主引导记录，占据磁盘的 0 磁道的第一个扇区
  • MBR 表内：用来载入操作系统的可执行代码，引导代码用来告诉计算机如何处理分区表，如何定位操作系统等等；分区用来管理硬盘的分区情况，是一个有效的引导记录扇区
MBR 占据 0 磁道的第一个扇区，占据了 512 字节：
  • 第一部分：预启动区：占用 446 字节
  • 第二部分：分区表，占用 64 字节
  • 第三部分：幻数 占用 2 字节，固定 55AA

常见的分区号：

  • 0x5 可扩占分区
  • 0x82 Linux 交换分区
  • 0x83 普通 Linux 分区
  • 0x8e Linux 逻辑卷管理分区
  • 0xfd Linux 的 raid 分区

为什么要创建分区：

  • 方便管理和控制
  • 提高系统的效率：分区后减少系统读写和磁盘的磁头移动距离；使用磁盘的分配额的功能区限制用户使用的磁盘容量；便于恢复和备份

5.2 fdisk 分区

  • MBR 内有一个占用 64 字节的分区表：保存的是分区表信息
  • 由于每一个分区会占用 16 个字节，最多可以做 4 个分区（四个主分区）
  • 为了解决分区不足的，就可以将分区表的 16 个字节存储另外一个信息（扩展分区）
  • 总结：硬盘的分区分为三种：主分区、扩展分区、逻辑分区

  • 它最少是有一个主分区，最多时有 4 个主分区
  • 三个主分区+一个扩展分区+若干逻辑分区
    • 如果没有扩展分区是不能创建逻辑分区
    • 扩展分区也不是真正存储数据的地方，逻辑分区才是存储数据的地方
  • 一个主分区+一个扩展分区和若干逻辑分区
    • 不管前面有多少个主分区或扩展分区，逻辑分区永远是从 5 开始的

六、文件系统结构

  • 磁盘可以被划分两个区域：一类是存储元数据区（inode），另一类就是存储真实数据的（date block）
    • inode 内划分了多个 inode block，每个大小 128b（字节）
    • data 划分多个 date block，每个大小为 4K（千字节）
  • 磁盘在存储数据的时候，至少是占用一个 inode 和一个 block
  • 如果要寻找的数据的时候，做全盘扫描的话，性能低下
    • inode bitmap 记录哪些 inode 被使用或者未被使用
    • date block bitmap 记录哪些 date block 被使用或者未被使用
  • 文件删除：删除目录下文件名称，然后把存储文件的 inode 和 date block 改成可被修改的状态，但是文件内容没有删除的，是可以恢复
  • 文件移动：只是将文件名称从一个目录移动到另一个目录，并不会修改 inode 和 block

九、Linux 逻辑卷管理机制（LVM）

  • 虽然有扩展硬盘的机制，经过分区、格式化之后，可以无限扩容，但是分区应该分多大？如何迁徙数据？
  • 如果在一开始就有管理机制帮助动态的管理存储，LVM 提高这种功能
  • LVM 最大的好处：可以随时调整分区的大小，分区内的时候不会丢失，而且不需要卸载分区、停止服务等等

9.1 LVM 的概念

LVM（逻辑卷管理机制）：在硬盘分区上建立一个逻辑层，这个逻辑层让多个硬盘或者多个分区看起来像是一个硬盘。

  • 真实的硬盘分区被称为物理卷（PV）：真实的物理硬盘或者硬盘分区
  • 由多个物理卷（PV）组成的逻辑上的硬盘，叫做卷组（VG）：可以使用同一个硬盘的不同分区，也可以是在不同硬盘的不同分区，最后显示的一个逻辑硬盘
  • 把卷组划分成多个可以使用的分区，就叫逻辑卷（LV）：卷组一个逻辑上的硬盘，需要使用一定要分区，分区就是 LV
  • 在 LVM 最小存储单位不在 block，是物理扩展（PE）：PE 是最小存储单位，大小是 4MB

十、RAID

10.1 RAID 是什么

RAID（磁盘阵列）：廉价由多个磁盘组成磁盘阵列。

  • 原理：通过软件或者硬件把多个比较小分区组合成一个容量比较大的磁盘组
  • 功能：提高读写性能、增加了数据冗余的功能

10.2 RAID 级别

10.2.1 RAID 0（带区卷）：RAID 级别内性能最好的

  • RAID 0 最好是由容量相同的两块或者两块以上的磁盘，如果容量不一致，会影响性能
  • 优点：读取和写入性能都是最快的，可使用容量也是最大的
  • 缺点：数据没有冗余，可靠性差
  • 磁盘容量：N * 磁盘数量
  • 磁盘利用率：100%
  • 适用场景：对读写性能要求高，但是不需要数据冗余功能

10.2.2 RAID 1（镜像卷）：至少是有两块磁盘组成，磁盘的容量一致，如果不一致容量是以小的那一块为主

  • RAID1 就具备了数据冗余功能，就是把一个写数据写进到两个磁盘
  • 优点：具备数据的冗余功能，提高了数据的安全性，允许坏一块磁盘
  • 缺点：可使用容量 50%，数据镜像；磁盘利用率低，成本高；磁盘写入性能一般

10.2.3 RAID5：最少是可以三块磁盘组成，磁盘容量一致是最好

  • 当使用 RAID 5 的时候，需要把硬盘分割成区块，当数据要写入磁盘的时候，数据也被划分了区块
  • 每次在循环写入数据的时候，在其中一个硬盘上加了奇偶校验值，校验值的内容就是这次循环写入其他磁盘数据备份
  • 优点：因为奇偶校验值的出现，raid5 有了数据冗余功能
  • 缺点：因为奇偶校验值需要损失一部分性能来进行生成和写入；RAID5 不管是多少磁盘组成，只支持一块磁盘损坏之后数据恢复；写的性能不高
  • 磁盘容量：（n-1）* 磁盘容量
  • 磁盘利用率：75%（四块磁盘组成 raid5）

其他级别：RAID 6、RAID1+0、RAID0+1

10.2.4 软 RAID 和硬 RAID 区别

  • 磁盘阵列卡（RAID 卡）组成 raid：性能高，不占用服务器的资源，唯一的缺点贵
  • 服务器搭建软 RAID，适合经费有限，但是又需要 raid 功能的，就可以使用软 raid

一、文件权限介绍

1.1 什么是权限

  • 权限是约束用户对系统的操作
  • 权限指某一个用户具有的特定系统资源试用权

1.2 为啥需要权限

  • Linux 是多用户系统，对于用户来说，个人隐私重要。所以要划分权限
    • 安全性、降低误删除的风险、减少故障和数据泄露
    • 数据隔离，不同权限看到的东西是不一样，操作也不一样
    • 职责明确

1.3 权限和用户的关系

  • Linux 内，权限是规定用户可以做什么、不能做什么
    • 针对文件定义了三种身份：属主（owner）、属组（group）、其他人（other）
    • 每个身份对应三个权限：read 读取（r）、write 写入（w）、execute 执行权限（x）
  • 用户访问流程：
    • 判断你是不是文件的属主，如果是就按照属主的权限进行访问
    • 判断你是不是文件的属组，如果是的话就按照属组的权限进行访问

  • 修改权限使用 chmod 命令实现：对于 root 用户来说，可以修改任何人的文件；对于普通用户而言，只能修改属于自己的文件（授权除外）
  • 修改权限方式：UGO 方式、NUM 方式

1.4.1 UGO 方式

  • 修改权限的单位：= 直接赋予权限（覆盖原有权限）、+ 在现在的基础上增加权限、- 在现在基础上减少权限

chmod  u+rwx 1.txt

chmod  u=rwx 1.txt

#设置属主权限为rwx，属组为r-x ，其他没有权限
chmod  u=rwx,g=rx,o= 1.txt

1.4.2 NUM 方式

#需要设置文件属主权限是r-x，属组是rwx，其他用户是r-x

三、特殊权限

3.1 SUID 特殊权限

3.1.1 SUID 生产背景：

  • 普通用户能不能改自己的密码？可以
  • 普通用户可不可以改其他用户的密码呢？不能
  • 存放密码信息的文件是哪个？/etc/shadow
  • 这个文件有什么权限？没有权限
  • 普通用户能不能更改这个文件？不可以
  • 那么现在为什么普通用户可以修改自己的密码呢？因为使用 passwd 命令。在 passwd 属主位置上本来应该 x 权限变成 S。s 是 suid，是设置属主。作用：在使用有 suid 文件或者命令的时候，会以属主的身份去执行
[root@localhost bin]# chmod  u+s cat    ugo添加权限s权限
[root@localhost bin]# ll cat
-rwsr-xr-x. 1 root root 54080 8月  20 2019 cat
[root@localhost bin]# chmod  u-s cat    ugo回收权限s权限
[root@localhost bin]# ll cat
-rwxr-xr-x. 1 root root 54080 8月  20 2019 cat

------------------------------------------------------
[root@localhost bin]# chmod  4755 cat    #num方式添加权限    s八进制数字是4
[root@localhost bin]# ll cat
-rwsr-xr-x. 1 root root 54080 8月  20 2019 cat

[root@localhost bin]# chmod  755 cat      #num方式回收权限    s八进制数字是4
[root@localhost bin]# ll cat
-rwxr-xr-x. 1 root root 54080 8月  20 2019 cat

3.1.2 总结：

  • 如果属主位上有 x 权限，再去添加 suid 的话会显示小写 s
  • 如果属主位上没有 x 权限，添加 suid 会显示大写 S
  • 让普通用户可执行的文件，临时拥有文件属主的权限
  • 特殊权限 suid 对二进制可执行程序有效=命令文件
  • 特殊权限 suid 不建议给 vi 或者 vim、rm

3.2 SGID 特殊权限

3.2.1 什么是 SGID：

  • SGID 设置二进制可执行程序，命令在执行的过程中会以命令属组的身份去执行
  • 如果你 SGID 设置目录上，在目录下创建子目录或者子文件都会继承父目录属组
sgid 设置在属组位置上，八进制数值是2

3.2.2 总结：

  • 如果属组位上有 x 权限，再去添加 SGID 的话会显示小写 s
  • 如果属组位上没有 x 权限，添加 SGID 会显示大写 S
  • 给目录添加 SGID 权限，添加之后新建的文件或者目录都会继承父目录的属组
  • SGID 权限可以让多个用户共享一个目录变得简单很多

四、文件默认权限

4.1 默认权限

  • 用户在创建文件或者是创建目录的时候，默认分配给文件或者是目录的权限

4.2 默认权限的由来

  • 背景：用户创建文件默认权限 rw-rw-rw 666；用户创建目录默认权限 rwxrwxrwx 777；实际情况文件 644 目录是 755
  • 分析：系统在创建文件或者是目录的时候会受到 UMASK 值的影响，默认是 022
    • 文件：666-022=644
    • 目录：777-022=755

4.3 系统默认权限配置文件

if [ $UID -gt 199 ] && [ "`/usr/bin/id -gn`" = "`/usr/bin/id -un`" ]; then
    umask 002
else
    umask 022
fi
  • 如果 UID 大于 199 则 umask 值是：002
  • 如果 UID 小于 199 则 umask 值是：022
  • 为什么 root 的 UMASK 是 022，这个是 /etc/profile 文件定义
  • 用户在登录系统的时候会加载 profile 文件进行判断

4.4 修改 umask

  • 永久修改
  • 临时修改（电脑重启或者重新加载就会失效）

五、提权

  • 在普通用户需要使用 sbin 目录下的命令时需要做一个提权
  • 需要提权的场景：普通用户没有足够的权限；root 权限太强应该少用；普通用户不得不使用特殊指令的时候
  • 提权的方式：
    • sudo：优点安全、方便；缺点需要提前制定规则，规则操作复杂
    • su：身份切换，相当于提权；优点方便；缺点需要知道提权者的密码

/root
[lisi@localhost root]$ exit

[root@localhost ~]# su

/home/lisi

5.1.2 shell 登录分类：

  • 登录式 shell 需要输入用户名、密码才能登录 shell
  • 非登陆式 shell 不需要用户名和密码就能进入 shell 的
  • 登录式和非登陆式加载环境变量文件顺序不一样：
    • 登陆式：/etc/profile > /etc/profile/*.sh > ~/.bash_profile > ~/.bashrc > /etc/bashrc
    • 非登录式：~/.bashrc > /etc/bashrc > /etc/profile/*.sh

5.1.3 环境变量配置：

  • profile 类文件：设定环境变量，登陆前运行脚本或者命令
  • 用户配置文件：~/.bash_profile、~/.bashrc
  • 全局环境变量：/etc/profile、/etc/profile/*.sh、/etc/bashrc

一、文件打包和压缩介绍

1.1 什么是压缩

  • 将多个文件或者目录合并成一个特殊文件

1.2 为什么要压缩

  • 存储或者传输大文件的时候，压缩可以减少大文件实际容量
  • 如果输出的时候有很多文件，这样子传输会降低效率，把多个文件压缩，再传输

1.3 不同系统的压缩格式互通

  • windows 的压缩包和 Linux 的压缩包互通？
    • 在 windows 系统内接触最多的压缩格式：zip、7z、gzip、rar
    • 在 Linux 系统内接触使用最多的压缩格式：zip、tar.gz
  • 一般需要互通的话，就选择：zip

1.4 Linux 常见的压缩包类型

  • .zip zip 压缩工具
  • .gz gzip 压缩工具，只能压缩文件，会删除源文件
  • .bz bzip2 压缩工具，只能压缩文件，会删除源文件
  • .tar.gz 使用 tar 命令先归档打包，然后使用 gzip 压缩
  • .tar.bz 使用 tar 命令先归档打包，然后使用 bzip2 压缩

2.1 gzip

  • gzip 压缩工具，只能压缩文件，会删除源文件

  • tar [参数] filename：

    • C 指定解压目录
    • X 排除多个文件
    • --exclude 排除文件或者目录
  • 常用的打包和压缩的组合命令：
    • czf 打包文件成 .tar.gz 格式
    • zcf 打包文件成 .tar.gz 格式

    • cJf 打包文件成 .tar.xz 格式
    • zxf 解压 tar.gz 包
    • jxf 解压 tar.bz2 包
    • xf 智能解压文件
    • tf 查询压缩包内的文件内容

重定向

▼

一、重定向

1.1 什么是重定向

  • 重定向就是改变默认的输入输出的方向
  • 默认情况下：标准输入是键盘；标准输出是显示器

1.2 为什么需要重定向

  • 命令的输出结果想要保存到文件中
  • 命令的输入需要从文件中读取
  • 过滤命令的输出

1.3 标准输入和输出

Linux 内一切皆文件，所以标准输入设备、标准输出设备被抽象成文件：

名称文件描述符设备说明
标准输入0键盘命令默认从键盘读取输入
标准输出1显示器命令默认把正确结果输出到显示器
标准错误2显示器命令默认把错误信息输出到显示器

二、输出重定向

  • 标准覆盖输出：> 把命令的正确结果输出到文件，会覆盖文件原有内容
  • 标准追加输出：>> 把命令的正确结果输出到文件，追加到文件末尾
  • 错误覆盖输出：2> 把命令的错误信息输出到文件，会覆盖文件原有内容
  • 错误追加输出：2>> 把命令的错误信息输出到文件，追加到文件末尾
  • 正确和错误都追加输出：&>>
  • 正确和错误都输出到同一个文件：> 文件 2>&1
#标准覆盖输出：把ls的结果输出到1.txt（会覆盖原有内容）
[root@localhost opt]# ls > 1.txt

#标准追加输出：把ls的结果追加到1.txt末尾
[root@localhost opt]# ls >> 1.txt

#错误覆盖输出：把错误信息输出到2.txt
[root@localhost opt]# ls /aaaa 2> 2.txt

#错误追加输出：把错误信息追加到2.txt末尾
[root@localhost opt]# ls /bbbb 2>> 2.txt

#正确和错误都输出到同一个文件
[root@localhost opt]# ls /opt /aaaa > 3.txt 2>&1
#或者
[root@localhost opt]# ls /opt /aaaa &> 3.txt

#正确和错误分别输出到不同文件
[root@localhost opt]# ls /opt /aaaa > 4.txt 2> 5.txt

#黑洞文件：丢弃输出内容
[root@localhost opt]# ls /aaaa 2> /dev/null

三、输入重定向

  • 输入重定向：< 把文件的内容作为命令的输入
#把1.txt的内容作为wc的输入

  • 输入重定向：<<EOF，把多行内容作为命令的输入

> EOF

四、管道符

  • 管道符 |：把前一个命令的输出作为后一个命令的输入
  • 管道符只能处理标准输出，不能处理标准错误

  • samba 是基于 smb 协议（server message block 服务信息块）的一个开源软件
  • 主要功能：让 Linux 系统和 windows 系统之间共享文件、打印机等资源

二、samba 服务器的组成

  • SMB（Server Message Block）：服务信息块协议，主要作用是网络文件共享
  • NBM（NetBIOS over TCP/IP）：通过 TCP/IP 协议栈承载 NetBIOS 名称解析，使得 windows 可以通过主机名访问 samba 共享

三、安装部署 Samba 服务

3.1 基础环境的搭建

  • 关闭防火墙
  • 关闭 selinux
#关闭防火墙
[root@localhost ~]# systemctl stop firewalld
[root@localhost ~]# systemctl disable firewalld

[root@localhost ~]# setenforce 0
[root@localhost ~]# sed -i 's#SELINUX=enforcing#SELINUX=disabled#g' /etc/selinux/config

3.5 修改配置文件

samba 配置文件路径：/etc/samba/smb.conf

[root@localhost ~]# vim /etc/samba/smb.conf

[global]
        workgroup = SAMBA
        security = user
        passdb backend = tdbsam
        printing = cups
        printcap name = cups
        load printers = yes
        cups options = raw

[share]
        comment = share
        path = /share
        browseable = yes
        public = yes
        writable = yes
  • [share] 共享名称，客户端访问时显示的名称
  • comment 共享描述
  • path 共享路径
  • browseable 是否可浏览
  • public 是否允许匿名访问
  • writable 是否可写

3.6 创建共享目录

[root@localhost ~]# mkdir /share

3.7 创建用户&设置密码

[root@localhost share]# useradd smb_root
[root@localhost share]# smbpasswd  -a smb_root
New SMB password:
Retype new SMB password:
Added user smb_root.

3.8 共享路径授权身份验证

[root@localhost share]# chmod  -R 775 /share
[root@localhost share]# chown  -R smb_root.smb_root /share
