一：计算机的原理

1.1 什么是计算机？

  • 计算机一般称为”电脑“通俗理解就是通电的大脑

  • 电脑：人类对计算机的最终期望

    • 希望他可以向人脑一样为人们工作
1.2 计算机的五大组成部分

  • 运算器

  • 控制器

  • 存储器

  • 输入设备

  • 输出设备

1.3 cpu

  • 控制器

    • 作用：计算机的指挥系统，负责管理计算机其他组件的工作

  • 运算器

    • 作用：包含数学运算、逻辑运算

  • 总结：控制器和运算器本质上不是硬件，抽象的概念

  • CPU就是安装控制器和运算器的标准制作的一个硬件，所以cpu又被称为计算机大脑

1.4 存储器

存储器主要是：内存和硬盘

  • 内存

硬盘

存储器I/O

  • 作用：负责数据的存取，有了存储器计算机才有记忆功能

  • 分类

    • 内存

      • 基于电工作，通电就能存取数据

        • 优势：存取速度很快

        • 劣势：断电之后数据会丢失

    • 硬盘（外存）

      • 基于磁工作

        • 优势：断电数据不会丢失，可以永久保存

        • 劣势：存取速度偏慢

输入设备

  • 向计算机内输入内容

  • 鼠标、键盘、麦克风

输出设备

  • 计算机向外输出数据

  • 显示器、音响、打印机

二、操作系统

2.1 操作系统的由来

  • 需求

    • 开发一个编辑工具，这个软件的核心就是文本编辑，但是在编辑的时候，需要牵扯到计算机的硬件

  • 不管编写什么软件，最终的目的就是操控硬件

  • 所有的硬件设备都是需要软件进行支配

  • 实现步骤

    • 第一步：必须要开发也控制系统，通过控制系统来控制计算机硬件基本运行

    • 第二步：然后开发编辑工具的功能，但凡涉及到了硬件，就调用控制系统

    • 第三步：最后由操作系统来控制计算机硬件

2.2 什么是操作系统

  • 控制系统别名：操作系统（OS）

    • 操作系统是在应用软件和硬件之间桥梁

    • 也是一个协调、管理、控制计算机硬件资源，软件资源的一个控制对象

    • 日常所使用的windows包括IOS系统或者安卓系统都是属于桌面或者个人系统

  • 服务器操作系统

2.3 Linux系统

  • 首先Linux和windows一样都是操作系统。但是Linux是由两层含义

    • 一种是Linux编写了开源操作系统的内核

    • 第二种就是广义上的操作系统

2.3 Linux 发展史
自由软件之父

  • 理查德.M.斯托曼发起了GNU组织

  • 主要项目

    • copyleft 无版权

    • copyright 有版权

    • opensource ：开放源代码，软件谁都可以使用，可以进行二次开发

    • free： 免费，未必开源

    • GPL： 通过版权许可证协议。

      • 如果软件打上了GPL，那么任何人都可以对他进行修改，但是修改完成之后必须将源码发布出来，开源为了更好的传承

  • Linux 相关软件大量采用 GPL 许可证

  • 自由软件运动的口号：团结就是力量

Linux 系统之父

  • Linus Torvalds在19岁的时候写出了的Linux内核

  • 操作系统的核心叫做内核，但是内核并不等于操作系统

  • 内核主要是提供系统级别的服务

    • 操作系统的构成：内核 操作系统框架 软件

    • 总结：内核是无法单独工作，必须要套上操作系统的框架，还要有GNU组织更正应用程序

  • 其实Linux官方叫GNU/Linux

Linux 系统的发行

  • Linux的发行主要分为两类：

    • 商业公司维护发行版本

    • 社区组织维护的发行版本

Redhat：企业级操作系统，Linux内核进行编译，安装相应的软件，测试之后发行
Ubuntu：社区维护，主要用于手机系统或者车载、电脑桌面系统
Centos： 社区企业级操作系统，redhat的开源版
为什么要使用centos

  • Centos兼具了社区和企业两种特点

  • Centos稳定、长期维护、开源免费、可扩展性强等等

3.3.1 缓存机制

  • 100000人访问访问服务器获取软件列表

  • 服务器需要响应十万次满足客户的需求

  • 如果每个空行一天操作几十几百次请求，会增加服务器的负载压力

  • 每次用户请求的时候，先看看本地是否有缓存列表，没有的话再向服务器请求

  • 服务器返回软件列表，客户端会列表信息缓存到本地

  • 再次请求先查看本地缓存信息

    • 缓存信息是有时间限制

一、文件打包和压缩介绍

1.1 什么是压缩
  • 将多个文件或者目录合并成一个特殊文件
1.2 为什么要压缩

  • 存储或者传输大文件的时候，压缩可以减少大文件实际容量

  • 如果输出的时候有很多文件，这样子传输会降低效率，把多个文件压缩，再传输

1.3 不同系统的压缩格式互通

  • windows的压缩包和Linux的压缩包互通？

    • 在windows系统内接触最多的压缩格式：zip、7z、gzip、rar

    • 在Linux系统内接触使用最多的压缩格式：zip、tar.gz

  • 一般需要互通的话，就选择：zip

1.4 Linux常见的压缩包类型

  • .zip zip压缩工具

  • .gz gzip压缩工具，只能压缩文件，会删除源文件

  • .bz bzip2压缩工具，只能压缩文件，会删除源文件

  • .tar.gz 使用tar命令先归档打包，然后使用gzip压缩

  • .tar.bz 使用tar命令先归档打包，然后使用bzip2压缩

一、文件权限介绍

1.1 什么是权限

  • 权限是约束用户对系统的操作

  • 权限指某一个用户具有的特定系统资源试用权

1.2 为啥需要权限

  • Linux是多用户系统，对于用户来说，个人隐私重要。所有要划分权限

    • 安全性、降低误删除的风险、减少故障和数据泄露

    • 数据隔离，不同权限看到的东西是不一样，操作也不一样

    • 职责明确

1.3 权限和用户的关系

  • Linux内，权限是规定用户可以做什么、不能作什么

    • 针对文件定义了三种身份

      • 属主 （owner）

      • 属组 （group）

      • 其他人 （other）

    • 每个身份对应三个权限

      • read 读取                简称：    r

      • write 写入 简称：w

      • execute 执行权限 简称：x

  • 用户访问流程

    • 判断你是不是文件的属主，如果是就按照属主的权限进行访问

    • 判断你是不是文件的属组，如果是的话就按照属组的权限进行访问

    • 如果判断你不是属主，也不是属组，就只能按照其他人的权限进行访问

第一列字符代表文件类型 第2到第4位代表是属主的权限 第5到第7代表的属组权限 第8-第10位代表的其他用户权限
d rwx r-x r-x

  • Linux内权限是 rwxr-x--- 9个字符组成

  • 主要控制文件的： 属主 属组 其他

  • 背景

    • 文件：666-022=644

    • 目录：777-022=755

if [ $UID -gt 199 ] && [ "`/usr/bin/id -gn`" = "`/usr/bin/id -un`" ]; then

else
    umask 022
fi

  • 用户在登录系统的时候会加载profile文件进行判断

一、用户的基本概述

1.1 什么是用户？

用户指的是能够正常登录Linux系统或者windows、mac os系统的账号，比如：QQ号、身份证号

1.2 为什么需要用户

  • 系统上运行的每一个进程，都是需要特定的用户才能运行

  • 通常在公司内使用普通用户进行管理，root权限太大了，容易造成破坏的

1.3 用户的分类
  • 用户管理就是去管理root用户创建出来的用户
用户ID 含义
0 超级管理员，最高权限，也有着极强的破坏力
1~200 系统用户，来运行系统自带进程（指令、软件等等），默认已经创建的
201~999 系统用户，用来运行用户安装的程序，此类无需登录系统
1000+ 普通用户，可以正常登录系统的用户，权限比较小，能执行的任务有限

  • 系统把用户的信息存放/etc/passwd文件

  • 系统把用户密码存放在/etc/shadow文件

  • 文件解析

  • /bin/bash 是咱们可以登录使用的

1.5.2 shadow文件

  • 文件解析

  • zhangsan:!!:19786:0:99999:7:::

zhangsan ！！ 19786 0 99999 7 空 空
用户名 密码字符串，!!代表没有设置密码 最近一次密码的变更时间 密码最小使用天限  0=无限 密码最长使用天数 99999=不过期 密码到期钱前，系统会在密码到期前7天提醒更改密码 密码到期后2天会强制用户换密码 账户失效时间

三、 用户组的管理

3.1 用户组概念

  • 组时逻辑层面的含义

  • 逻辑上将多个用户归纳到一起，当对组进行管理的时候，就是对组内所有用户进行管理

  • 共享组的权利

3.2 组的分类

  • 针对用户来说，分为以下几类

    • 默认组：你在创建用户的时候，如果不指定组，那么默认会创建一个和用户同名的组

    • 基本组：用户只能由一个基本组，创建的时候-g 指定

    • 附加组：用户可以有多个附加组，创建时候-G指定

3.3 组相关配置文件

  • /etc/gshdow 保存组密码

  • 将原本要输出到屏幕上面的信息，重定向到某一个文件内容或者定向到其他
1.2 为什么需要重定向

  • 程序在执行输出的信息比较多，运行的打印状态信息也很多，可以把这个打印的信息输出重定向到一个日志文件进行保存

1.3 标准输入和输出

  • 当进程操作一个文件的时候

    • 首先程序无法之间访问硬件，需要Linux内核帮助

    • 内核kernel利用文件描述符来访问文件

  • 总结：进程》通过文件描述符》访问》文件名称

    • 进程就是使用文件描述符管理打开的文件对应关系

  • 通常程序访问一个文件至少会打开三个文件：分别是标准输入、输出、错误输出

文件名称（filename） 3+

二、 磁盘介绍

2.1 什么是磁盘

硬盘是电脑永久存储数据的硬件：文件、视频、音频等等

每个硬盘中心都是一摞告诉运转的圆盘，在圆盘上附着一圈金属颗粒，每个金属颗粒都会有自己的磁化程度，主要存储0和1

2.2 磁盘物理结构

2.2.1 什么盘片

硬盘一般是由一个或者多个盘片组成的，每个盘片都是可以有两面，打开一个盘片正面0面，背面1面后面盘片以此类推

2.2.2 磁道

盘片在出厂的时候被划分了多个不同直径圆环，数据就存储在这样圆环，圆环又被称为磁道（track）

2.2.3 扇区

硬盘出厂的时候修磁盘进行磁道划分，划分成若干个弧段，每个弧段都被称为扇区，扇区也是硬盘上的物理存储单位，每个扇区可以存储512字节

2.2.4 柱面

柱面是抽象的逻辑上的概念，简单说：出于同一个垂直一个取悦的磁道被称为柱面。

作用：主要为了通过算法检测磁头的寻磁时间

2.2.5 磁头

读取磁盘上金属颗粒，主要负责读取或者写入数据

2.3 磁盘的接口类型
2.3.1 IDE-SCSI

比较旧的硬盘接口

2.3.2 SATA-SAS
  • SATA III 和 SAS
2.3.3 MSATA-M2

  • MSATA 接口主要是为超极本设计的

  • M2是inter推出的一种条带MSTATA接口规范

四、 分区管理

4.1 为什么要分区

Linux系统初始化的时候，跟MBR识别硬盘设备

MBR:硬盘主引导记录，占据磁盘的0磁道的第一个扇区

MBR表内：用来载入操作系统的可执行代码，引导代码用来告诉计算机如何处理分区表，如何定位操作系统等等；分区用来管理硬盘的分区情况，是一个有效的引导记录扇区

  • MBR占据0磁道的第一个扇区，占据了512字节

    • 第一部分：预启动区：占用446字节

    • 第二部分：分区表，占用64字节

    • 第三部分：幻数 占用2字节，固定55AA

  • 常见的分区号

    • 0x5 可扩占分区

    • 0x82 Linux交换分区

    • 0x83 普通Linux分区

    • 0x8e Linux逻辑卷管理分区

    • 0xfd Linux的raid分区

为什么要创建分区

  • 方便管理和控制

  • 提高系统的效率

    • 分区后减少系统读写和磁盘的磁头移动距离

    • 使用磁盘的分配额的功能区限制用户使用的

    • 磁盘容量

    • 便于恢复和备份

    • 三个主分区+一个扩展分区+若干逻辑分区

      • 如果没有扩展分区是不能创建逻辑分区

      • 扩展分区也不是真正存储数据的地方，逻辑分区才是存储数据的地方

    • 一个主分区+一个扩展分区和若干逻辑分区

      • 不管前面有多少个主分区或扩展分区，逻辑分区永远是从5开始的

六、文件系统结构

  • 磁盘可以被划分未两个区域：一类是存储元数据区（inode），另一类就是存储真实数据的（date block）

    • inode内划分了多个inode block，每个大小未128b（字节）

    • date划分多个date block，每个大小为4K（千字节）

  • 磁盘在存储数据的时候，至少是占用一个inode和一个block

  • 如果要寻找的数据的时候，做全盘扫描的话，性能低下

    • inode bitmp 记录那些inode被使用或者未被使用

    • date block bitmp 记录那些date block被使用或者未被使用

  • 文件删除

    • 删除目录下文件名称，然后把存储文件的inode和date block改成可被修改的状态，但是文件内容没有删除的，是可以恢复

  • 文件移动

    • 只是将文件名称从一个目录移动到另一个目录，并不会修改inode和block

  • LVM最大的好处：可以随时调整分区的大小，分区内的时候不会对视，而且不需要卸载分区、停止服务等等

9.1 LVM的概念

LVM（逻辑卷管理机制）：在硬盘分区上建立一个逻辑层，这个逻辑层让多个硬盘或者多个分区看起来像是一个硬盘

  • 真实的硬盘分区被称为物理卷（PV）

    • 真实的物理硬盘或者硬盘分区

  • 由多个物理卷（PV）组成的逻辑上的硬盘，叫做卷组（VG）

    • 可以使用同一个硬盘的不同分区

    • 也可以是在不同硬盘的不同分区

    • 最后显示的一个逻辑硬盘

  • 把卷组划分成多个可以使用的分区，就叫逻辑卷（LV）

    • 卷组一个逻辑上的硬盘，需要使用一定要分区，分区就是LV

  • 在LVM最小存储单位不在block，是物理扩展（PE）

    PE是最小存储单位，大小是4MB

十、RAID

10.1 RAID 是什么

RAID（磁盘阵列）：廉价由多个磁盘组成磁盘阵列。

  • 原理：通过软件或者硬件把多个比较小分区组合成一个容量比较大的磁盘组

  • 功能：提高读写性能、增加了数据冗余的功能

10.2 RAID级别
10.2.1 RAID 0

  • RAID 0 (带区卷)：RAID 级别内性能最好的

  • RAID 0 最好是由容量相同的两块或者两块以上的磁盘，如果容量不一致，会影响性能

  • 特点：

    • 优点：读取和写入性能都是最快的，可使用容量也是最大的，

    • 缺点：数据没有冗余，可靠性差；

    • 磁盘容量：N*磁盘数量

    • 磁盘利用率：100%

    • 适用场景：对读写性能要求高，但是不需要数据冗余功能

10.2.2 RAID 1

  • RAID1 镜像卷，至少是有两块磁盘组成，磁盘的容量一致，如果不一致容量是以小的那一块为主

  • RAID1 就具备了数据冗余功能，就是把一个写数据写进到两个磁盘

  • 特点

    • 优点：具备数据的冗余功能，提高了数据的安全性，允许坏一块磁盘

    • 缺点：可使用容量 50%，数据镜像；磁盘利用率低，成本高

      • 磁盘写入性能一般
10.2.3 RAID5

  • 最少是可以三块磁盘组成，磁盘容量一致是最好

  • 当适用RAID 5 的时候，需要把硬盘分割成区块，当数据要写入磁盘的时候，数据业被划分了区块

  • 每次在循环写入数据的时候，在其中一个硬盘上加了奇偶校验值，校验值的内容就是这次循环写入其他磁盘数据备份

  • 特点：

    • 优点

      • 因为奇偶校验值的出现，raid5 有了数据冗余功能

    • 缺点：

      • 因为奇偶校验值需要损失一部分性能来进行生成和写入

      • RAID5 不管是多少磁盘组成，只支持一块磁盘损坏之后数据恢复

      • 写的性能不高

      • 磁盘容量：（n-1）*磁盘容量

      • 磁盘利用率：75%（四块磁盘组成raid5）

RAID 6 RAID1+0 RAID0+1

10.2.4 软RAID和硬RAID区别

  • 磁盘阵列卡（RAID卡）组成raid：性能高，不占用服务器的资源，唯一的缺点贵！

  • 咱们要讲解的就是服务器搭建软RAID，适合经费有限，但是由需要raid功能的，就可以使用软raid

一、samba简介

  • samba是一个可以让linux系统应用和网络通讯的软件

  • 功能：可以在Linux和windows系统之间文件共享或者打印共享，

  • samba也是一个windows和Linux之间的文件共享协议

二、 samba服务器的组成

  • SMB

    • SMB是samba的核心启动程序，负责建立Linux samba服务器和samba客户机之间的对话，验证客户身份提供文件和打印系统的访问。

    • 只要启动SMB服务之后，才能实现文件共享

    • 监听端口：139tcp端口

  • NBM

    • NBM是负责解析，类似DNS解析功能

    • NBM可以把Linux系统内共享各种名称、组名和IP对应起来

    • 如果没有搭建NBM，就只能提供IP地址去访问

    • 监听端口：137和138 UDP端口

1.2 vim的使用
1.2.1 vim的三种模式

  • 普通模式：进入文件内的第一个界面就是普通模式，不能编辑而只能使用光标进行移动定位

  • 编辑模式：通过按键” a A I i O o“进入编辑模式，文件左下角会出现”插入“字样，此时就可以对文件进行编辑，按ESC 即可退回普通模式

  • 末行模式：

    • 普通模式：按 ： 或者是 / 进入末行模式

    • 在末行模式下可以进行的操作：保存、退出、搜索、替换、显示行号

1.2.2 普通模式的使用

  • 命令光标跳转

    • G 光标跳转到文件末行

    • gg 光标会跳转到文件首行

    • Ngg 光标跳转到指定行，N是数字

    • $ 光标跳转到当前光标所在行的行尾

  • 文件内容多翻页显示

    • ctrl + f 向下翻页

    • ctrl + b 向上翻页

  • 复制粘贴

    • yy 复制当前所在的行

    • Nyy 复制几行，N是数字

    • P 粘贴到光标所在的下一行

    • P 粘贴到光标所在的上一行

  • 删除、撤销、反撤销

    • dd 删除当前光标所在的行

    • Nd 删除当前光标所在行以及向下几行

    • dG 删除当前光标所在行以及后面的所以行

    • u 撤销上一步操作

    • ctrl + r 反撤销
