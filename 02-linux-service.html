1.1 什么是网络

  • 网络由网络连接设备通过传输介质将网络终端设备连接起来进行数据交换、资源共享

  • 网络连接设备：路由器、交换机

  • 传输介质：

    • 有线传输介质：网线、光纤、同轴电缆

    • 无线传输介质：卫星、微波、无限电波等

  • 网络终端设备：计算机、服务器、手机、智能电视等等

1.1.1 工作方式

  • 边缘部分：用户直接使用

  • 核心部分：为边缘部分服务

1.1.2 计算机网络的组成

  • 硬件：主机、双绞线、光纤、通信设备等等

  • 软件：安装在系统上的应用软件

  • 协议：一系列规则、约定的集合

1.2 OSI七层模型

物理层：网卡

数据链路层：交换机

more 命令

more [选项] 文件名

  • -f ：计算行数时，以实际行数为主，而不是自动换行后的行数

  • -p ： 不以卷动的方式显示每一页内容，而是先清楚屏幕然后显示内容

  • -c ：和-p功能像是，不同的地方是先显示内容再去清除

  • -s ：当连续遇到两个空行以上，就会替换成一个空行

  • -u ：不显示引号

  • +n ：从第几行开始显示文件内容

  • -n：一次显示行数

  • ctrl+f ：向下翻页

案例：

一次显示6行
[root@localhost ~]# more -6 /opt/1.txt

从第六行开始显示
[root@localhost ~]# more +6 /opt/1.txt

  • NF：当前行呗分割的列数，NF代表的是最后一个字段，NF-1代表倒数第二个

[root@localhost opt]# awk '{print FILENAME,NR,NF,$O }' awk.txt

PXE

1.1 什么是pxe

pxe（预启动执行环境）

pxe是intel公司推出的一款通过网络来引导操作系统的协议，广泛应用于网吧无盘系统

只需要把无盘服务器维护好，就能保证网络中无盘工作站（电脑）的系统正常运行

PXE服务器需要支持软件：DHCP、tftp、syslinux（pxe引导的程序文件） 文件共享（samba、nfs）

1.2 PXE服务器和客户机的工作流程

  1. pxe客户机发送DHCP请求，向服务器申请IP地址

  2. DHCP服务器响应PXE客户机的请求，自动从IP地址池分配IP地址给客户机，并告知TFTP服务器IP地址和pxw引导程序pxelinux.0

  3. PXE客户机向tftp服务器发起获取pxelinux.0引导文件

  4. TFTP服务器响应PXE客户机的请求，将其共享的pxelinux.0引导程序给pxe客户机

  5. pxe客户机通过网络来启动安装界面

  6. pxe客户机向samba发起获取操作系统文件请求

  7. samba（文件共享服务器）响应pxe客户机的请求，将共享文件系统传输个pxe客户机

  8. pxe客户机进行安装提示相对，客户需要手动解决安装

实际安装的时候，可以采用kickstart应答文件，来进行无人值守安装

Cobbler

2.1 什么是cobbler

Cobbler是一个快速网络安装Linux的服务，现在也可以通过自定义脚本网络安装windows

利用一些简单的命令就可以完成pxe网络安装环境的配置，同时还可以管理DHCP、DFS、TFTP、rsync以及yum仓库

2.2 cobbler的作用

  • 服务器上架后，可以手动选择需要安装的操作系统

  • 服务器上架后，可以根据需求安装配置操作系统

  • 系统安装后，可以执行自定义脚本，安装系统基础软件初始化

2.3 kickstart 是什么

  • kickstart是redhat（红帽）开发的一个工具，可以理解为：一个自动安装应答配置管理程序

  • kickstar安装选项，包含：语言、防火墙、密码、网络分区等等配置

1.2 FTP的工作原理

FTP是采用互联网标准文件传输协议，给用户提供一组管理计算机之间文件传输的应用程序

1.3 传输方式
1.3.1 ASCLL

  • 假设用户拷贝的文件包含了简单ASCLL码文本，就算你远端的计算机他不是Linux系统，当文件传输的时候，ftp通过会自动调整文件的内容，方便把文件解析为另一台点卡可以存储的文本格式

  • ASCLL更通用，效率没有二进制高

1.3.2二进制
  • 在二进制传输的时候，保存文件的位序，以便原始和拷贝的逐位一一对应，即使目的机器上包含位序的文件是没有任何意义，所以基本是不能互通

五、FTP工作模式

5.1 主动模式

  • 工作原理

    • FTP客户端连接FTP服务器开放的21端口,发送用户名、密码登录

    • 客户端随机开放一个端口（1024以上），把随机端口给到服务器

    • 服务器接收到随机端口后，会使用20号端口连接客户端开放的随机端口

5.2 被动模式

  • 工作原理

    • FTP客户端连接FTP服务器开放的21端口,发送用户名、密码登录

    • 客户端发出PASV命令，告知服务器被动模式，控制连接建立成功

    • 连接成功之后，服务器会生成一个随机端口，将这个端口告知给客户端

    • 客户端主动连接服务器随机端口，建立数据连接

5.3 如何选择

  • 如果你们是在一个局域网，或者你在一个计算机上，可以使用主动模式

  • 如果是在公网，比如阿里云服务器建议使用被动模式

1.1 什么DHCP

DHCP（动态主机配置协议），是一个局域网的网络协议使用UDP协议

作用：给内外主机自动分配IP地址，给用户或者内部的网络网络管理员作为管理所有计算机的手段

DHCP是C/S模型：DHCP客户端和服务器要保持通信，DHCP是基于UDP协议，双方都有可能主动向对方发起通信，因此双方都需要监听端口

  • server端：67/UDP

  • client端：68/UDP

1.2DHCP的优势

  • 减少管理员工作量

  • 避免输入错误的可能性

  • 避免IP地址冲突

  • 更改IP网段的时候，不需要手动重新分配每一个IP地址

1.3 DHCP的分配方式

  • 手动分配：由DHCP服务器管理指定IP地址

  • 自动分配：分配IP地址之后，永久使用

  • 动态分配：使用完成之后会自动释放IP，供其他用户使用

1.4 DHCP的工作原理

  • 客户端请求IP地址

    • 客户端启动之后，本身是没有IP地址，通过广播的方式找dhcp服务器

  • 服务器响应

    • 服务器在收到请求，会在自己的IP地址池查找是否有合法，如果查找到之后会给IP做个记号加入到dhcp offer内发送给客户端

  • 客户机选择IP地址

  • 服务器确认租约

  • 函数的参数是shell位置参数的一种，在函数在可以使用$n进行接收

    • $n 位置参数

    • $# 传递给脚本或者函数的参数的个数

    • $@ 传递给脚本或是函数所有的参数

    • $* 传递给脚本或是函数所有的参数

  • 案例

#! /bin/bash

function a(){
  echo "name: $1"
  echo "age: $2"
  echo "add：$3"
  echo "我是$1,家住$3,今年$2岁"
}

a "关羽" "48" "蜀国"

19.7 函数的返回值

  • shell中这个返回值代表函数的退出状态，返回0就是成功了，非0就是失败

  • 返回值的范围：0-255之间的整数

  • 执行失败时，可以定义退出状态的值，每一个状态代表不同的错误

  • 如果幻术内return 语句，那么使用默认退出状态，也就是最后一个命令的退出状态，为了严谨的写法 return $?

[root@localhost opt]# cat 5.sh
#! /bin/bash
#### 做函数的返回值

function say(){
  local sum=0
  for ((i=1;i<=2;i++))
  do
    ((sum+=i))
  done
  echo $sum
  return $?
}

TCP和UDP协议

TCP协议

  • 面向连接：在传输数据的时候，要建立客户端和服务器的连接，才能传输数据

  • 可靠的通信：TCP传输数据的时候，会基于内部的各种机制保证数据传输到目的地

  • 基于字节流：TCP传输的是基于字节传输，易于对数据的拆分和合并发送

UDP协议

  • 无连接：只需要知道目的地IP和端口号就可以发送数据，无需建立连接

  • 不可靠的通信：没有一系列机制应对传输时丢包的问题

  • 面向数据包发送：应用层给UDP什么报文，UDP就会发送什么

  • 适用范围：由于UDP是不属于连接性协议，所以小号资源比较小，处理速度优的特点，因此适用于视频、音频通话的时候

TCP三次握手

  • 客户端发送一个SYN（同步标识包）给服务端

  • 服务器接收到SYN包后，回传一个SYN/ACK的数据包传递确认信息，表示我收到了

  • 客户端再回传一个ACK标志的数据包，表示我知道，握手结束

TCP四次挥手

  • 客户端发送一个FIN，用来关闭客户端到服务端的数据发送

  • 服务端在收到FIN的时候，会返回一个ACK给客户端

  • 服务端发送一个FIN给客户端，用来关闭服务端到客户端的数据发送

  • 客户端收到FIN后会返回一个ACK给服务端，确认断开
