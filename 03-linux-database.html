MySQL 数据库安装（移除MariaDB / 配置源 / 首次登录 / 远程访问）

▼

第一步：移除默认安装的 MariaDB

# 查看已安装的mariadb
rpm -qa | grep mariadb
# 输出示例: mariadb-libs-5.5.68-1.el7.x86_64

# 卸载mariadb
yum remove -y mariadb-libs-5.5.68-1.el7.x86_64

第二步：配置 MySQL 8.x 安装源

# 导入MySQL官方密钥
rpm --import https://repo.mysql.com/RPM-GPG-KEY-mysql-2023

# 配置MySQL yum仓库
rpm -Uvh https://dev.mysql.com/get/mysql80-community-release-el7-11.noarch.rpm

第三步：安装 MySQL

yum -y install mysql-community-server

第四步：启动服务

systemctl start mysqld
systemctl enable mysqld
systemctl status mysqld

第五步：首次登录

# 查看临时密码
grep 'temporary password' /var/log/mysqld.log

# 使用临时密码登录
mysql -u root -p
# 输入临时密码
# Server version: 8.0.36

# 如果临时密码失效，使用以下方案：
# 1. 编辑配置文件
vim /etc/my.cnf
# 在 [mysqld] 下方添加：
# skip-grant-tables    # 忽略密码验证

# 2. 重启服务
systemctl restart mysqld

# 3. 无密码登录
mysql -u root -p    # 直接回车

# 4. 刷新权限并设置密码
mysql> flush privileges;
mysql> alter user 'root'@'localhost' identified by 'Abc123456!';

# 5. 去掉配置文件中的 skip-grant-tables，重启服务
systemctl restart mysqld

第六步：修改登录密码

# 密码要求：英文大小写 + 数字 + 特殊符号，至少8位
mysql> alter user 'root'@'localhost' identified by 'Abc123456!';

第七步：配置远程访问

# 创建远程登录用户
mysql> create user 'root'@'%' identified with mysql_native_password by 'Bbc123456!';

# 授予远程登录权限
mysql> grant all on *.* to 'root'@'%';

# 验证远程登录
mysql -uroot -h 192.168.10.128 -pBbc123456!

# 查看数据库列表（远程登录后）
mysql> show databases;
+--------------------+
| Database           |
+--------------------+
| information_schema |
| mysql              |
| performance_schema |
| sys                |

    -> ('喜羊羊',18,'羊村一号','羊村守护者','123'),

mysql> delete from wh where age<10;

DQL - 数据查询

# 检索所有信息
mysql> select * from wh;

# 检索指定列
mysql> select username,age from wh;

# 统计类查询
mysql> select count(*) from wh;                    # 总行数
mysql> select min(age) from wh;                    # 最小值
mysql> select max(age) from wh;                    # 最大值
mysql> select sum(age) from wh;                    # 总和
mysql> select avg(age) from wh;                    # 平均值（排除null）
mysql> select count(*),min(age),max(age),sum(age),avg(age) from wh;

# 模糊查询
mysql> select * from wh where username like '张%';  # %通配符
mysql> select * from wh where username like '___';  # _占位符

# 空查询
mysql> select * from wh where age is null;           # 过滤为null
mysql> select * from wh where age is not null;       # 过滤不为null
mysql> select * from wh where age is null or age='';  # 两种空状态

# 运算符
# 算数: () * / + - %
# 比较: < <= > >= <> !=(5.6+) =
# 逻辑: not and or
mysql> select * from wh where age>10 and age<30;

# 排序查询
mysql> select * from gn order by age desc;            # 降序
mysql> select * from gn order by iq asc, age asc;     # 多条件排序

连接查询

# 内连接 - 只有对应的数据才显示
mysql> select * from class,student where class.cid=student.cid;

# 内连接（JOIN写法）
mysql> select class.cname, student.sname
    -> from class
    -> inner join student on class.cid = student.cid;

# 左外连接 - 以左表为主，无匹配显示NULL
mysql> select class.cname, student.sname
    -> from class
    -> left join student on class.cid = student.cid;

# 右外连接 - 以右表为主，无匹配显示NULL
mysql> select class.cname, student.sname
    -> from class
    -> right join student on class.cid = student.cid;

索引

索引类型特点创建方式
主键索引唯一 + 非空 + 自动排序，一张表只能一个primary key / alter table add primary key
唯一索引值唯一，允许空值，一个表可多个unique / create unique index
普通索引无特殊限制，仅加速查询index / create index
组合索引多个列组合，遵循最左前缀原则create index idx_name on table(col1,col2)
 
# 查看索引
mysql> show index from class;

# 主键索引
mysql> alter table a add primary key(cid);
mysql> alter table a drop primary key;

# 唯一索引
mysql> create unique index un_iq on a(iq);
mysql> drop index un_iq on a;

# 普通索引
mysql> create index I_name on a(name);
mysql> alter table a add index index_pass(name);
mysql> drop index I_name on a;

# 组合索引
mysql> create index index_group on a(name,iq);
mysql> drop index index_group on a;

# 创建索引原则
# 1. 表数据超过300条应建索引
# 2. 唯一性差的字段不建议建索引
# 3. 更新频繁的字段不适合建索引
# 4. 经常出现在where子句中的字段应建索引

主从复制搭建部署（IO/SQL 线程原理 / 异步半同步复制 / 部署步骤）

▼

主从复制工作原理

主从复制流程：

1. 主库需要开启二进制日志（binlog），并授权给从库访问
2. 从库开启一个 IO 线程，与主库建立连接，读取主库 binlog 日志内容，
   将读取到的日志写入到 relay.log（中继日志）
   如果主从数据一致，IO线程休眠，直到主库产生新内容才同步
3. 从库开启一个 SQL 线程，读取 relay.log 中的内容并在从库执行，
   从而实现主从复制

主从复制类型

类型说明优缺点
异步复制（默认）主库执行完毕后直接返回结果给客户端，不等待从库效率高但容易出现数据不一致
同步复制主库等待所有从库复制且执行成功后才返回结果数据一致但效率和性能受影响
半同步复制主库等待至少一个从库写入relay日志后返回结果兼顾性能和数据安全

复制模式：基于语句复制（默认）、基于行的复制、混合类型复制。

环境准备

数据库备份管理（MyISAM vs InnoDB / 温备份 mysqldump / 冷备份 tar / 增量备份 binlog）

▼

为什么要备份

备份的目的：解决数据库的灾难恢复，实现容灾。

数据丢失原因：开发人员bug、操作失误、天灾人祸、设备老化、黑客攻击等。

备份方式分类

分类方式说明
物理备份冷备份关闭数据库进行备份：tar打包
物理备份热备份数据库正常运行时备份：hotcopy
物理备份温备份数据库锁定时备份：mysqldump
逻辑备份完全备份全部数据备份
逻辑备份差异备份备份与上次备份不同的数据
逻辑备份增量备份备份上次备份后新增的内容

MyISAM vs InnoDB 对比

特性MyISAMInnoDB
默认版本MySQL 5.5之前MySQL 5.5之后
事务支持不支持支持（ACID）
锁定级别表级锁定行级锁定
外键不支持支持
读写阻塞读写相互阻塞读写分离不阻塞
缓存只缓存索引索引和数据都可缓存
存储文件.frm(结构) + .MYD(数据) + .MYI(索引).frm(结构) + .ibd(索引+数据)
适用场景以读为主、少量更新、无事务要求频繁更新、有事务要求的场景
 
# 查看支持的引擎
mysql> show engines;

# 查看当前默认引擎
mysql> show variables like '%storage_engines%';

# 查看某个表的引擎
mysql> show table status from abc where name='class';

# 修改表的存储引擎
mysql> alter table class engine=MyISAM;

# 创建表时指定引擎
mysql> create table asd(NAME int) engine=MyISAM;

# 临时修改默认引擎
mysql> set default_storage_engine=MyISAM;

# 永久修改默认引擎
vim /etc/my.cnf
# [mysqld]
# default_storage_engine=MyISAM
systemctl restart mysqld
# 注意：修改不会对已存在的表生效

温备份 - mysqldump

# mysqldump将库/表导出为SQL脚本，恢复时重新执行

# 备份单个库
mysqldump -uroot -p --databases abc > /opt/abc.sql

# 备份多个库
mysqldump -uroot -p --databases abc cateye1 > /opt/abc-cateye.sql

# 备份所有库
mysqldump -uroot -pAbc123456! --all-databases > /opt/all.sql

# 备份指定表
mysqldump -uroot -pAbc123456! abc gn > /opt/gn.sql

# 还原数据库（在MySQL内执行）
mysql> drop database abc;
mysql> source /opt/abc.sql;

# 还原所有库
mysql -uroot -p < /opt/all.sql

冷备份 - tar

# 确定数据目录
cat /etc/my.cnf | egrep datadir
# datadir=/var/lib/mysql

# 打包备份（含日期）
tar zcvf /opt/mysql$(date +%F).tar.gz /var/lib/mysql

# 模拟数据丢失
mkdir /opt/mysql
mv /var/lib/mysql/* /opt/mysql

# 还原数据
mkdir linshi
tar zvxf mysql2024-04-22.tar.gz -C linshi/
mv /opt/linshi/var/lib/mysql/* /var/lib/mysql
systemctl restart mysqld

热备份 - cp

# 拷贝源文件
cp -a /var/lib/mysql/* /opt/mysql

# 模拟数据丢失
mv /var/lib/mysql/* linshi/

# 还原数据
cp -a mysql/* /var/lib/mysql/
systemctl restart mysqld

增量备份 - binlog

增量备份基于完全备份，需要开启二进制日志。

# 1. 先做完全备份
mysqldump -uroot -pAbc123456! --all-databases > /opt/mysql_all.sql

# 2. 开启二进制日志
vim /etc/my.cnf
# 添加：log-bin=mysql-bin
systemctl restart mysqld

# 3. 同步时间
yum -y install ntpdate
ntpdate ntp1.aliyun.com

# 4. 生成新的日志文件（开始增量记录）
mysqladmin -uroot -pAbc123456! flush-logs

# 5. 查看日志内容
mysqlbinlog /var/lib/mysql/mysql-bin.000001

# 6. 基于时间点恢复
# 恢复到指定时间点（执行该时间前的所有命令）
mysqlbinlog --no-defaults --stop-datetime='2024-04-23 11:17:57' \
  /var/lib/mysql/mysql-bin.000003 | mysql -uroot -pAbc123456!

# 从指定时间点开始恢复
mysqlbinlog --no-defaults --start-datetime='2024-04-23 11:34:16' \
  /var/lib/mysql/mysql-bin.000003 | mysql -uroot -pAbc123456!

# 限制时间段恢复（同时指定start和stop）
