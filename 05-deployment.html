服务器 - Nginx 部署

▼

HTTP 服务器

接受 web 请求、响应 web 请求，在功能上还实现模块化结构设计，支持通用语言接口：例如 PHP\Perl\Python 等语言；同时还支持正向代理和反向代理、URL 重写、压缩传输、ssl 加密等等。

Apache（httpd）（80）

是一个开源跨平台的服务器，也称未 httpd，他是 Apache 基金会品牌产品。

Nginx（80）

  • Nginx 也是开源，但是有个别功能是收费
  • 它也是伊戈尔在 2004 年发布的 web 服务器，现在大多都是使用 nginx 作为 web 服务器

特点：

  • 高性能、轻量级 web 服务器软件
  • 跨平台使用
  • 模块化设计、扩展性好
  • 稳定性高
  • 支持热部署，不统计更新配置文件
  • 系统资源消耗低
  • 对 http 并发连接处理速度高

Apache 和 Nginx 的区别

  • apache 请求处理模式：是把每一个请求作为一个新线程进行处理
    • 多线程可以增加软件运行效率
    • 缺点：多线程会消耗更多资源
  • Nginx 使用事件驱动：一个线程处理多个请求，并且使用异步
    • 没有使用多线程，避免资源占用
    • 使用异步，提高吞吐量
  • 静态文件处理能力：Nginx 的处理能力是 Apache 的三倍以上
  • 资源消耗：Nginx 低于 Apache
  • 负载能力：Nginx 高于 Apache
  • 配置文件：Nginx 简单
  • 热部署：Nginx 支持热部署
问题与解决方案：
IP、端口号是直接暴漏出去的；无法完成协作（两个猫眼程序不能互相协作）。
解决方案：Nginx 反向代理、负载均衡。
  • 反向代理：中介代理房东，用户相当于租客；nginx 代理猫眼应用，用户访问时通过访问 Nginx 得到数据，黑客看到的只是 Nginx 提供的网址和端口。
  • 负载均衡：中介可以代理多个房主，Nginx 也可以代理多个服务器。轮询、加权轮询、最小轮询等。如果 A 服务器坏掉了，B 服务器可以承担压力。

部署 Nginx（开一个新的虚拟机）：利用 Nginx 修改配置文件，让 Nginx 可以代理 192.168.10.128:8888 猫眼（反向代理）；利用 Nginx 代理多个，做负载均衡。

2.2 反向代理

安装目的：让用户在访问 192.168.10.129:80 看到 192.168.10.128:8888 对应的数据。

修改配置文件：

[root@localhost nginx]# cd /usr/local/nginx/conf/
[root@localhost conf]# cp nginx.conf{,.bak}
    server {
        listen       80;
        server_name  localhost;

        #charset koi8-r;

        #access_log  logs/host.access.log  main;

        location / {
         #   root   html;
         #   index  index.html index.htm;
         proxy_pass_header Server;
         proxy_set_header Host $http_host;
         proxy_set_header X-Real-IP $remote_addr;
         proxy_pass http://192.168.10.128:8888;
        }


[root@localhost conf]# cd ../sbin/
#检查配置文件
[root@localhost sbin]# ./nginx -t
nginx: the configuration file /usr/local/nginx/conf/nginx.conf syntax is ok
nginx: configuration file /usr/local/nginx/conf/nginx.conf test is successful


[root@localhost sbin]# ./nginx  -s reload

验证反向代理：除了可以访问 192.168.10.128:8888/api/films/?currentindex=2&cateld=5，还可以使用 192.168.10.129:80/api/films/?currentindex=2&cateld=5。

2.3 反向代理 + 负载均衡

[root@localhost sbin]# vim /usr/local/nginx/conf/nginx.conf
    upstream cateye{    #声明一个服务组，里面定义要代理的服务IP：端口
        server 192.168.10.128:8888;
        server 192.168.10.128:8889;
    }

    server {
        listen       80;
        server_name  localhost;

        #charset koi8-r;

        #access_log  logs/host.access.log  main;

        location / {
         #   root   html;
         #   index  index.html index.htm;
         proxy_pass_header Server;
         proxy_set_header Host $http_host;
         proxy_set_header X-Real-IP $remote_addr;
         proxy_pass http://cateye;    #代理的是服务站
        }
