Zabbix 监控介绍

▼

监控系统简介

Zabbix 是一个基于 Web 界面提供的分布式系统监控，可以提供网络监控的功能，是一个企业级开源监控套件。

Zabbix 的监控对象

  • Linux、Windows 等等主机系统
  • 路由器、交换机等网络设备

Zabbix 的版本

  • Zabbix 是 2001 年发布的首个 GPL 版本
  • 官方在后期不断的维护、升级、性能完善等等
  • 目前最新的 zabbix 6.*
  • 目前市场大范围使用的：5.* LTS 版本
    • LTS：长期维护
    • GPL：开源

Zabbix 五大组件

组件端口说明
Zabbix Server10051Zabbix 的核心组件，server 内部存储了所有配置信息、统计信息、操作信息
Database-Zabbix 数据库内存储配置信息、统计信息、操作信息
Zabbix Web-Web 界面也是 Zabbix 的一部分，一般和 Zabbix Server 部署在一个机器上（特殊情况可分开）。Web 页面提供直观的监控信息，方便维护管理
Zabbix Agent10050部署在被监控端，主要用于主动监控本地资源和应用程序，把监控到的数据发送给 Zabbix Server
Zabbix Proxy10051Proxy 可根据生产环境决定是否采用。架构较大可采用 Proxy；企业设备跨网段、跨机房，Server 和 Agent 无法直接连接时可采用 Proxy

Zabbix 的工作原理

  • Zabbix Server 通过 Zabbix Agent 获取被监控的信息
  • Server 计算信息是否满足触发条件来判断是否给用户发通知
  • Server 收取到信息都会存储到数据库内
  • 数据库给 Zabbix Web 提供信息，Web 生成操作界面，显示相应的信息

Zabbix 监控的两个模式

特性主动模式被动模式
定义客户端主动向服务器建立连接，发送数据服务器指定向客户端请求数据
工作流程Agent 每隔一段时间，主动向 Server 发起连接请求；Server 接收到请求后，查询客户端需要的信息，发送给客户端，由客户端授权数据发送给服务器客户端需要开启端口 10050，等待服务器来收集数据，客户端把收集到的数据发送给服务器
端口Server 端口 10051（Agent 连入）Agent 端口 10050（Server 连入）
主动模式使用场景：当服务器监控的主机到达了一定的数量的时候，就可以让 Server 收集信息。

Zabbix 提供两种安装方式：官网打包下载（https://www.zabbix.com/cn/download）和命令式安装。以下为命令式安装。

[root@zabbix-server ~]# rpm -Uvh https://mirrors.aliyun.com/zabbix/zabbix/5.0/rhel/7/x86_64/zabbix-release-5.0-1.el7.noarch.rpm

Web 界面手动添加主机

在 Zabbix Web 界面中修改网页语言为中文后，手动添加主机：

  • Configuration → Hosts → Create host
  • 填写 Host name、选择 Templates、配置 Agent interfaces（IP + 10050 端口）
  • 手动添加监控模板（策略）
如果监控失败了，排查思路：
手动添加主机的时候，IP、名称都核对对应了吗；在选择监控模板的时候，是否选择出错；始终是红色，IP\名称、防火墙、内核，agent（10050）端口正常运行。

自动发现主机

▼

网络主动发现

  • Zabbix 会周期性的扫描：网络发现规则定义 IP 网段
  • 网络发现主要由两个部分组成：发现和动作

3.1 准备一个部署完成的被监控端

被监控端的准备基础环境、安装 Zabbix 源、安装 zabbix-agent、修改配置文件、启动等步骤与"手动添加主机"完全一致（Server=192.168.10.128、ServerActive=192.168.10.128、Hostname=host1）。

监控端和被监控端时间同步

yum -y install ntpdate
[root@zabbix-server ~]# ntpdate ntp1.aliyun.com

3.2 创建发现规则

在 Zabbix Web 界面创建发现规则：

  • Configuration → Discovery → Create discovery rule
  • 配置 IP range（扫描网段）、Delay（发现频率）
  • Checks：Check type = Zabbix agent，Port range = 10050
关键点：Zabbix 服务器会自动扫描地址段内的所有 IP，通过 system.uname 键值查询是否可以获取数据，如果可以获取数据，就会把主机添加到规则内。

3.3 添加动作

在 Zabbix Web 界面添加触发条件以及处理动作：

  • Configuration → Actions → Discovery actions → Create action
  • 配置 Action 名称、Conditions（发现规则 + 服务类型）
  • 配置 Operations：Add host、Add to host group、Link to templates、Enable host

至此自动发现部署完成：配置自动发现规则，添加触发条件以及处理动作。

自动注册

▼

自动注册原理

  • Zabbix Server 等着 Zabbix Agent 来上报
  • 缺点：有时候 Agent 找不到 Server
  • 自动注册和自动发现是相反的

一、基础环境

教师机 IP: 192.168.10.140
配置 DNS：8.8.8.8 或者 114.114.114.114
举例：
DNS1=8.8.8.8

关闭防火墙和内核机制
setenforce 0         # 临时关闭
vi /etc/selinux/config    # 永久关闭
SELINUX=disable

systemctl stop firewalld
systemctl disable firewalld

设置完成后重启
